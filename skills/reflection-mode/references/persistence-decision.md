# Persistence Decision

Reflection Mode（反思模式）区分“反思方法”和“反思产物”。不是每条经验都应该保存，但达到写入标准的经验也不应该长期停留在候选状态。

默认行为是先判断写入等级：不沉淀、候选待确认、自动写入。保存长期记忆、skill reference 或其他持久化产物，必须遵守当前宿主的 memory 和文件系统规则；当经验值得自动写入且宿主允许时，应直接写入。

## 自动写入标准

同时满足以下条件时，推荐自动写入：

- 经验基于清晰证据，不是猜测。
- 它会改变未来判断、验证步骤或执行方式。
- 它稳定且有复发可能，未来重新发现成本较高。
- 目标位置清楚：长期记忆、skill reference 或其他明确产物。
- 当前宿主允许对应写入，且用户没有表达反对。

不满足这些条件时，退回候选待确认或不沉淀。

## 长期记忆决策

当经验是稳定事实、用户偏好、仓库约定、流程细节或重复出现的项目特定模式，并且应该影响未来对话时，优先考虑长期记忆。若同时达到自动写入标准且宿主允许，就写入长期记忆；否则给出长期记忆候选。

适合作为长期记忆候选的情况：

- 用户反复要求某种回复风格或工作流。
- 仓库有不明显的验证或发布命令。
- 项目存在反复踩坑、重新发现成本高的陷阱。
- 用户纠正暴露出稳定偏好。

如果经验只是猜测、过于宽泛、偏情绪化，或未来任务中很难复用，不要推荐为长期记忆。

## Skill Reference 决策

当经验是可复用方法、决策框架、输出契约或领域流程，并且能指导多类未来任务时，优先考虑 skill reference。若同时达到自动写入标准且宿主允许，就写入 reference；否则给出 skill reference 候选。

适合作为 reference 候选的情况：

- 一套反思流程。
- 一个沉淀决策框架。
- 一个 review 评判标准。
- 一个可重复的项目初始化或发布流程。
- 一种结构化输出格式。

Reference 应该教 Agent 如何思考或行动，不应该保存一次性任务历史。

## 不沉淀决策

以下情况推荐不生成持久化产物：

- 普通推理已经显然能得出。
- 只和当前对话有关。
- 基于不完整证据。
- 是一次性偶发事件，复发概率低。
- 只是感受，没有对应行为改变。

## 决策问题

推荐或保存前，先回答：

1. 它会改变未来动作或判断吗？
2. 它足够稳定，能经受上下文变化吗？
3. 长期记忆、reference、候选待确认、不沉淀，哪个是更小、更清晰的处理方式？
4. 自动写入是否比等待确认更能减少重复工作或重复错误？
5. 当前宿主是否允许这种写入？

如果答案不清楚，就把它保留为反思记录，不要变成持久规则。

## 写入边界

每次反思都遵守这个边界：

- 如果用户只要求反思，先完成反思和写入等级判断；若达到自动写入标准且宿主允许，可以写入并说明写入内容，否则返回建议或候选。
- 如果用户要求保存，优先执行写入；证据或目标位置不足时，先给出候选文本和目标位置。
- 具体写入时，只保存可复用经验，不保存完整反思叙事，并且只能通过当前宿主支持的 memory 或文件系统机制执行。
- 如果没有可用写入机制或权限，返回候选并说明尚未保存。
- 如果候选仍带有猜测性质，标记为 provisional，不要做成稳定规则。
