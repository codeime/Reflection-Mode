# Reflective Practice

Reflective Practice（反思实践）把过去经验转化为更好的未来行为。它适用于任务完成后、用户纠正后、同类错误重复出现时，或 Agent 发现自己被某个先入假设影响结果时。

## 反思方法

反思不是套固定模板，而是把一次任务中的证据转化为未来可复用的判断。先确认事实和偏差，再提炼经验、适用边界和验证信号；证据不足时保留不确定性，不要急着写成永久规则。

1. 事件：发生了什么？
2. 预期：原本应该发生什么？
3. 偏差：现实在哪里偏离了预期？
4. 未解问题：还有什么不知道，不能过早下结论？
5. 候选原因：偏差可能为什么出现？
6. 候选经验：可以抽象出什么可复用原则？
7. 未来触发：下次什么时候应该激活这条经验？
8. 沉淀建议：它应该被推荐为长期记忆候选、skill reference 候选，还是无需保存？
9. 验证信号：以后如何知道这条经验真的产生了帮助？

## 证据来源

- 用户原始请求和后续纠正。
- Agent 的假设、计划、命令、编辑或工具选择。
- 实际输出、测试结果、失败信息或 review 发现。
- 跨任务或跨项目重复出现的模式。
- 任何会改变未来行为的明确用户偏好。

## 经验质量

好的经验应该：

- 具体到能指导未来动作。
- 抽象到不只适用于单次事件。
- 基于证据，而不是情绪。
- 足够小，便于记住和复用。
- 明确未来触发条件。

弱经验通常是：

- 没有行动含义的流水账。
- 从一次特殊事件推出的万能规则。
- 对任务结果的重复描述。
- 类似“以后更小心”这种模糊目标。

## 反思深度

低风险任务或经验很明显时，做轻量反思。同类失败重复出现、用户纠正了 Agent、工作影响共享资产，或未来重复犯错成本较高时，增加证据、边界和沉淀判断的细节。
